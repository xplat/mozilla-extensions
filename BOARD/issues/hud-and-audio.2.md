The HUD needs to get its own file instead of being sprayed all over the place (although still mostly in viewer-media-playable.js).  After that we need an orderly notion of attaching a new element to the HUD or detaching one if it is still attached, to be used by playable content objects.  And we need to separate concerns between updating the HUD and saving persistent state information, and between playing/pausing/stopping the *visible* media and the *audible* media.
